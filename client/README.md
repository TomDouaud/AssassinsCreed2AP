# AC2AP client (.asi)

Archipelago client for Assassin's Creed II, loaded by an Ultimate ASI Loader
(e.g. a `winmm.dll`/`dinput8.dll` proxy) already present in the game folder.

## How it works â€” save-file detection

- Watches `1.save` (by mtime) and parses the completion records
  (`src/records.hpp`, a faithful port of the lab tooling, checked against ground
  truth by `test_records`).
- New records â†’ maps the game ID â†’ AP location via `AC2AP_map.txt` â†’
  `LocationChecks` to the server (apclientpp, non-TLS WebSocket).
- Received items are applied in-game where implemented (florins, traps,
  DeathLink); pure logical progression items are just marked applied.
- Persistent state (`AC2AP_seen.txt`, `AC2AP_pending.txt`, `AC2AP_items.txt`,
  `AC2AP_counted.txt`) survives reloads â†’ no double-sends, no lost checks while
  the server is down.

Detection is file-based and does not depend on RAM addresses, so it is robust
across launches. The RAM hooks (health/notoriety) are opt-in via
`enable_health_hook` and only needed for traps and DeathLink.

## Build

Prerequisites: VS 2022 (MSVC x86), CMake. Dependencies are vendored under `deps/`.

```
cmake -B build -A Win32 -DAC2AP_WITH_AP=ON
cmake --build build --config Release
```

Produces `build/Release/AC2AP.asi` (32-bit) and `test_records.exe` (parser
self-test).

**TLS / OpenSSL (required)** â€” archipelago.gg rooms are WSS-only; without TLS the
client can only reach plain `ws://` servers (e.g. localhost) and webhosted rooms
fail with "End of file". OpenSSL x86 static comes from vcpkg, vendored too:

```
git clone --depth 1 https://github.com/microsoft/vcpkg deps/vcpkg
deps/vcpkg/bootstrap-vcpkg.bat
deps/vcpkg/vcpkg.exe install openssl:x86-windows-static
```

CMake links it statically (`libssl.lib` + `libcrypto.lib` + `crypt32`) and fails
with a clear message if it's missing. Server certificates come from the Windows
cert store (wincrypt), so nothing extra ships with the asi.

Notes:
- asio `master` breaks websocketpp (`io_service` was removed) â€” use the pinned commit.
- zlib is required by websocketpp's `permessage_deflate` and is compiled in.
- key defines: `ASIO_STANDALONE AP_NO_SCHEMA _WEBSOCKETPP_CPP11_THREAD_` (do NOT
  define `WSWRAP_NO_SSL` â€” that was the webhosted-room bug)
- `project(AC2AP CXX C)`: C must be enabled to compile zlib's `.c` files

## Deploy (dist/)

Copy into the game's ASI scripts folder (next to the loader):
- `AC2AP.asi`
- `AC2AP.ini` (save_path, server, slot, password, enable_health_hook)
- `AC2AP_map.txt` (regenerated by `gen_map.py` from the apworld)

Runtime log: `AC2AP.log` next to the `.asi`.

## Debug command file

While running, the worker polls `AC2AP_cmd.txt` (same folder) for a one-line dev
command, executes it, then deletes the file. Useful for testing traps/DeathLink
and for RAM reverse-engineering (`sinit`/`sfilter`/`sdump` differential scanner,
`find`, `villa`, `money`, `kill`, ...). Arguments are parsed base-0 (decimal or
`0x` hex).
